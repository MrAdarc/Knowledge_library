# 说明
&#8195;&#8195;本文档的主要内容在于对相应的功能，完成相关的理论设计、实验设计、施工执行、优化测试等几个环节，为了便于进行项目管理，这里使用markdown文档进行工程管理，以记录在工程当中的测试、调试过程：

```
环节一：理论设计与实验设计
环节二：施工执行与功能实现
环节三：架构与精度优化测试
```
&#8195;&#8195;整个工程记录样板采用单个markdown文件进行记录，使用atom编辑器和markdown preview enhanced插件便于进行图像绘制和分析，同时便于进行插入图片、代码、表格等。

&#8195;&#8195;在markdown文件中，从理论设计到最后的优化测试，根据这种顺序对应的内容的特点，使用目录分级的结构进行记录，不同级别的标题分别为下述含义：
|标题级别|含义|
|---|---|
|一级标题|整体工程的理论设计|
|二级标题|功能模块的理论设计|
|三级标题|功能模块的实验设计|
|四级标题|施工执行及功能测试|
|五级标题|功能模块的架构优化|
|六级标题|功能模块的精度优化|
